### Especificação de Caso de Uso: Visualizar Dashboard

**Identificador:** UC-06

**Ator Principal:** Usuário **Requisitos Relacionados:** RF-007 (Dashboard de Saldos)
**Objetivo:** Apresentar ao usuário uma visão consolidada e centralizada de seus ativos (milhas/pontos), exibindo o somatório total de pontos acumulados agrupados por cada Programa de Fidelidade cadastrado.

#### 1. Pré-condições

- O Usuário deve estar devidamente autenticado no sistema.

#### 2. Condição de Entrada (Gatilho)

- O Usuário realiza o login no sistema (sendo redirecionado automaticamente para esta tela) ou clica no ícone "Dashboard/Início" no menu de navegação.

#### 3. Fluxo Principal (Caminho Feliz)

Este fluxo descreve o carregamento e exibição dos saldos consolidados.

1. **[Ator]** acessa a tela inicial do sistema.
2. **[Sistema]** consulta a base de dados para identificar todos os Programas de Fidelidade vinculados ao usuário.
3. **[Sistema]** realiza o processamento de cálculo (agrupamento): soma o valor de pontos de todas as transações registradas para cada programa específico.
4. **[Sistema]** gera os cartões (cards) visuais contendo o "Nome do Programa" (ex: Livelo) e o "Saldo Total de Pontos" (ex: 50.000).
5. **[Sistema]** exibe a tela do Dashboard com os totais organizados.
6. **O caso de uso termina** (o usuário visualiza a informação).

#### 4. Fluxos Alternativos (Variantes) UC-06

Cenários onde não há dados para processar.

##### **FA-01: Dashboard Vazio (Primeiro Acesso)**

- **Gatilho:** No passo 2 do Fluxo Principal.

1. **[Sistema]** verifica que não existem programas ou transações cadastradas para o usuário.
2. **[Sistema]** exibe uma mensagem de boas-vindas ou um estado vazio (_empty state_): "Nenhum ponto registrado ainda. Comece cadastrando seus programas e cartões.".
3. **[Sistema]** exibe atalhos rápidos (botões) para "Cadastrar Programa" ou "Cadastrar Cartão".
4. **O caso de uso termina.**

#### 5. Fluxos de Exceção (Tratamento de Erros)

Falhas no carregamento das informações.

##### **FE-01: Erro de Processamento/Conexão**

- **Gatilho:** No passo 2 ou 3 do Fluxo Principal.

1. **[Sistema]** falha ao tentar conectar ao banco de dados ou realizar o somatório dos pontos.
2. **[Sistema]** exibe uma mensagem de erro genérica na tela: "Não foi possível carregar seu saldo no momento. Tente novamente mais tarde.".
3. **[Retorno]:** O sistema mantém a estrutura da página carregada, mas sem os valores, permitindo que o usuário navegue para outras áreas.

### 6. Pós-condições

- O usuário obtém a informação atualizada sobre o total de pontos que possui em cada programa.
- Nenhuma alteração é feita no banco de dados (operação somente de leitura).
