### Especificação de Caso de Uso: Validar Dados (Abstrato)

**Identificador:** UC-08

**Tipo:** Caso de Uso de Inclusão (Internal/Abstract) **Requisitos Relacionados:** RF-008 (Validação de Dados)
**Objetivo:** Centralizar as regras de consistência e integridade dos dados, impedindo que informações inválidas (nulas, negativas ou incoerentes) sejam persistidas no banco de dados.

#### 1. Pré-condições

- Um caso de uso chamador (ex: Registrar Transação, Manter Cartões) deve ter coletado dados de entrada do usuário.

#### 2. Condição de Entrada (Gatilho)

- Ocorre automaticamente quando o usuário submete qualquer formulário de cadastro ou edição (Chamado pelos casos UC-02, UC-03, UC-04 e UC-05).

#### 3. Fluxo Principal (Processo de Validação)

1. **[Sistema]** recebe o conjunto de dados enviado pelo formulário.
2. **[Sistema]** verifica **Campos Obrigatórios**: Confere se algum campo mandatório está vazio ou nulo.
3. **[Sistema]** verifica **Tipos de Dados**: Confere se campos numéricos contêm apenas números e se datas são válidas.
4. **[Sistema]** verifica **Regras de Negócio Genéricas**:
   - Valores monetários não podem ser negativos ou zero.
   - Datas de transações não podem ser futuras (se aplicável).
   - Limites de cartão devem ser positivos.

5. **[Sistema]** confirma que os dados estão "Limpos".
6. **O caso de uso termina com sucesso** (retornando o controle para o caso de uso chamador prosseguir com a gravação).

#### 4. Fluxos de Exceção (Falha na Validação)

##### **FE-01: Inconsistência Detectada**

- **Gatilho:** Qualquer regra dos passos 2, 3 ou 4 é violada.

1. **[Sistema]** compila a lista de erros encontrados (ex: "O campo Valor é obrigatório").
2. **[Sistema]** retorna o status de **ERRO** para o caso de uso chamador.
3. **[Sistema]** interrompe a tentativa de gravação no banco de dados.
